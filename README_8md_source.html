5 # Features # {#features}
6 
7 * Header only. Boost licensed.
8 * Depends only on Boost headers & C++14 standard library. [note 3](#notes)
9 * Fully compatible with documented BSON specs.
10 * A strict, iterator-based JSON parser. Mostly compliant with JSON standard. [note 2](#notes)
11 * Parses [MongoDB extended JSON](http://docs.mongodb.org/manual/reference/mongodb-extended-json/) from UTF-8,16,32 strings.
12 * User-defined JSON literals `""_json_set`, `""_json_doc`, `""_json_arr`.
13 * An implementation of [JSONPath](http://goessner.net/articles/JsonPath/), with some support for filtering with boolean expressions.

19 ## Elements {#elements}
20 
21 `jbson::basic_element` is the class through which elements' names & values are accessed and has a template parameter for its data container. Generally, this will be a non-owning range (e.g. `boost::iterator_range`), but there is an `jbson::element` alias to `jbson::basic_element<std::vector<char>>` to enforce data ownership. The names are *always* copied (may change in the future). Value data may or may not be, depending on the container template parameter.
22 The type of an element can be determined through comparison with the `jbson::element_type` enum class values.
23 Values can be accessed through the `jbson::basic_element::value()` member function or the `jbson::get()` free function.
24 
25 ~~~cpp
26  using jbson::element_type;
27  jbson::element elem;
28  //...
29  if(elem.type() == element_type::string_element) {
30  auto str = elem.value<std::string>();
31  //OR
32  auto str = jbson::get<std::string>(elem);
33  //OR if container is contiguous
34  auto str = elem.value<boost::string_ref>();
35  //OR
36  auto str = jbson::get<boost::string_ref>(elem);
37  //OR returns either std::string or boost::string_ref, depending on container
38  auto str = jbson::get<element_type::string_element>(elem);
39  }
40 ~~~
41 
42 Elements can be modified using the `value()` member function.
43 
44 ~~~cpp
45  using jbson::element_type;
46  jbson::element elem;
47  elem.value("some string");
48  assert(elem.type() == element_type::string_element);
49  elem.value(123);
50  assert(elem.type() == element_type::int32_element);
51  elem.value<bool>(123);
52  assert(elem.type() == element_type::boolean_element);
53  elem.value<element_type::boolean_element>(123);
54  assert(elem.type() == element_type::boolean_element);
55  elem.value(element_type::boolean_element, 123);
56  assert(elem.type() == element_type::boolean_element);
57 ~~~
58 
59 Elements can also be accessed via the visitor pattern.
60 
61 ~~~cpp
62  using namespace jbson;
63  element elem;
64  //...
65  struct Visitor {
66  template <typename T>
67  void operator()(boost::string_ref name, element_type etype, T&& value) const {
68  // do something
69  }
70 
71  void operator()(boost::string_ref name, element_type etype) const {
72  // for elements whose type cannot have a value
73  // e.g. null_element, undefined_element
74  }
75  };
76  elem.visit(Visitor{});
77 ~~~
78 
79 Visitors can also return values. It is not necessary to typedef a `return_type`.
80 
81 ~~~cpp
82  using namespace jbson;
83  element elem;
84  //...
85  struct IsNull {
86  template <typename T>
87  bool operator()(boost::string_ref, element_type, T&&) const {
88  return false;
89  }
90  bool operator()(boost::string_ref, element_type) const {
91  return true;
92  }
93  };
94  bool null = elem.visit(IsNull{});
95 ~~~

97 ## Building Documents & Arrays {#building}
98 
99 `builder` & `array_builder` are simple classes used for building documents & arrays, respectively.
100 These allow for documents & arrays to be built from variables, rather than just literals.
101 
102 ~~~cpp
103  using namespace jbson;
104  auto str = "str"s;
105  int num = 123;
106  // {
107  // "some string": "str",
108  // "some int": 123,
109  // "some obj": {
110  // "child bool": false
111  // }
112  // }
113  document doc = builder
114  ("some string", element_type::string_element, str)
115  ("some int", element_type::int32_element, num)
116  ("some obj", element_type::document_element, builder
117  ("child bool", element_type::boolean_element, false)
118  );
119 ~~~
120 
121 While the intent of this is clear, it's a little verbose due to the explicit type.
122 This can be omitted when the type is compatible with a JSON type.
123 The above example becomes:
124 
125 ~~~cpp
126  using namespace jbson;
127  auto str = "str"s;
128  int num = 123;
129  // {
130  // "some string": "str",
131  // "some int": 123,
132  // "some obj": {
133  // "child bool": false
134  // }
135  // }
136  document doc = builder
137  ("some string", str)
138  ("some int", num)
139  ("some obj", builder
140  ("child bool", false)
141  );
142 ~~~
143 
144 That's about as good as it's going to get. `array_builder` is used the same way, apart from the lack of a name parameter, which is an implicitly set integer.
145 
146 ~~~cpp
147  using namespace jbson;
148  auto str = "str"s;
149  int num = 123;
150  // ["str", 123, { "child bool": false }]
151  array arr = array_builder
152  (str)
153  (num)
154  (builder
155  ("child bool", false)
156  );
157 ~~~

159 ## Documents {#docs}
160 
161 The main classes of jbson are the templates `jbson::basic_document`, `jbson::basic_array` & `jbson::basic_element`. These types are parameterised with their underlying container or range. It's generally recommended to use a random-access range. Their default aliases i.e. `document` & `element` are parameterised with `std::vector<char>`. The second template parameter of `basic_document` defines the parameter of child `basic_element`s. By default this is defined as a `boost::iterator_range` to avoid copying data, though care should be taken that this element does **not** outlive the container/document which owns the data, else it can be copied to its own `jbson::element`.
162 
163 ~~~cpp
164  jbson::document doc;
165  //...
166  for(auto&& element: doc) {
167  //...
168  }
169  std::for_each(doc.begin(), doc.end(), [](auto&& v) { something(v); });
170 ~~~
171 
172 `basic_array` is implemented using `basic_document` and the classes can usually be interchanged freely. Of course, `basic_document` is for documents and `basic_array` is for arrays and should be used as such.
173 
174 `basic_document` is simply a constant wrapper around its BSON data container, allowing iteration of elements in the form of a `basic_element`. It can be used with C++11's range-based for loop, and non-modifying standard algorithms.
175 Iterators retain a copy of elements. This does not mean that an iterator contains a copy of an element's data, however.
176 
177 ## Document Set {#doc_set}
178 
179 `basic_document_set` is an alias template to `std::set<basic_element,...>` which `basic_document` and `basic_array` are explicitly convertible to. This can be used to easily modify an existing document, at the cost of copying/converting the entire document into its constituent elements. The type parameter of `basic_document_set` should be a *container* not a *range*, i.e. it should own the data. The default alias `document_set` is `basic_document_set<std::vector<char>>` making it a set of `basic_element<std::vector<char>>`.
180 
181 ~~~cpp
182  jbson::document doc;
183  //...
184  jbson::document_set set(doc);
185  //... modify set
186  doc = set; // convert back
187 ~~~

189 ## JSON Parsing {#json}
190 
191 JSON documents can be parse with the `json_reader` class, which parses directly to BSON data and can be implicitly converted to any valid `basic_document`, `basic_array` or `basic_document_set`. It can also be move-converted to a `document` or `array` for efficient conversion.
192 
193 ~~~cpp
194  jbson::json_reader reader{};
195  reader.parse("{\"some json\": 123 }");
196  reader.parse(R"({"some json": 123 })"); // C++11 raw string
197  auto doc = jbson::document(std::move(reader));
198 ~~~
199 
200 `json_reader` can parse string literals, or any range of UTF-(8, 16 or 32) characters e.g. `std::string`, `boost::string_ref`, `boost::iterator_range<...>`, `QByteArray`, etc. To parse UTF-16 or UTF-32 documents, `json_reader` must be passed a range of (or iterators to) `char16_t` or `char32_t`, respectively. It is assumed the input code units use the system endian, and a BOM (byte-order mark) at the beginning of input is an error. `json_reader` may also work with `wchar_t`, though this usage is non-standard, non-portable and unsupported.
201 For convenience, user-defined literals have been implemented for JSON documents.
202 
203 ~~~cpp
204  using jbson::literal;
205  auto doc = R"({
206  "some json": 123
207  })"_json_doc;
208 ~~~
209 
210 This is equivalent to the above example.

257 # Notes {#notes}
258 
259 1. jbson uses BSON-like terminology. Eg. a document is not only the root object, but also all embedded objects.
260 2. BSON is incompatible with JSON in that element names cannot contain '\0', making names not fully UTF-8 compliant, which JSON requires.
261 3. Requires codecvt header (C++11, missing in gcc-4.8 & gcc-4.9 stdlib), various template aliases (C++14), probably more where gcc's stdlib is behind on the standard.
262 Known only to work with an up-to-date libc++ currently.
